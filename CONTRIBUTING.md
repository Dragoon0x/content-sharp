# Contributing

Contributions welcome. To contribute:

1. Fork the repository
2. Create a feature branch
3. Follow the existing skill format (frontmatter + structured content)
4. Ensure content is original, practitioner-depth, and framework-oriented
5. Submit a pull request with a description of what you're adding and why

Quality bar: every skill should contain named frameworks, specific examples, and actionable guidance. Generic advice is not accepted.
